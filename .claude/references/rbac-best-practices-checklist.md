# RBAC 最佳实践检查清单

## 核心模型
- [x] 用户 ↔ 角色 ↔ 菜单（权限）三层模型
- [x] 菜单支持三级结构：目录 → 菜单 → 按钮
- [x] 按钮级别关联权限标识（permission）
- [x] 角色支持内置（不可删除）和自定义两种类型

## 权限校验
- [x] 后端使用 @PreAuthorize + SpEL 注解式校验
- [x] 超级管理员直接放行（不走菜单匹配）
- [x] 权限校验走 Redis 缓存（命中率 > 99%）
- [x] 缓存变更时即时清除（角色/菜单/用户角色变更）

## 数据权限
- [x] 数据权限绑定角色，不绑定用户
- [x] 支持 5 种数据范围（ALL / DEPT_CUSTOM / DEPT_ONLY / DEPT_AND_CHILD / SELF）
- [x] 多角色数据权限取并集（最大权限原则）
- [x] SQL 自动拦截，业务代码无感知

## 安全防护
- [x] 内置角色不可删除
- [x] 删除角色/菜单/用户时级联清理关联表
- [x] 角色禁用后权限立即失效
- [x] 权限标识全局唯一
- [x] 前后端双重权限校验

## 多租户兼容
- [x] 角色表租户隔离（每租户独立角色）
- [x] 角色菜单关联表租户隔离
- [x] 菜单表全局共享（@SkipTenantFilter）
- [x] 租户套餐控制可用菜单范围
